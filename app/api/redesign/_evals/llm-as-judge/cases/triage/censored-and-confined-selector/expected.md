This session contains three AOIs.

## AOI 1

When entering a new task in the input field at the top of the list, typed
characters appear as dots or asterisks rather than the actual text. The user
cannot see what they are typing and has no way to verify or correct typos
before submitting the task. The underlying cause is that the input is
configured as a password-type field. Any wording along the lines of "input is
masked", "typed text is hidden as dots/asterisks", "user can't see what they
typed", or "input behaves as a password field" identifies the same AOI.

## AOI 2

The user attempts to mark tasks complete by clicking the task text itself
(for example, clicking directly on "Buy groceries" multiple times), but
nothing happens. Only the small checkbox icon actually toggles the task's
completion state. The click target is confined to the checkbox — the row and
text label look interactive but aren't. Any wording along the lines of "task
text isn't clickable", "checkbox is too small a target", "row/label
affordance is unclear", "toggle target is confined to the checkbox", or
"user has to click the exact checkbox to toggle" identifies the same AOI.


## AOI 3

After the user submits a new task via the input at the top of the list, the
newly-created item appears at the BOTTOM of the list rather than at the top
next to the input. The user has to scroll all the way down to confirm the
task was added.
